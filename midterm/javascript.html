Mutator Methods

Modifies the target array

Array.prototype.copyWithin()

Gets part of an array to another location on the array. And returns the array (with the same length).

Syntax:

arr.copyWithin(target)

arr.copyWithin(target, start)

arr.copyWithin(target, start, end)

Example:

var sample = ['ichi' 'san', 'yon', 'go', 'ni'].copyWithin(2, 0);

console.log(sample);

// Result : ["ichi", "san", "ichi", "san", "yon"]

Array.prototype.pop()

Removes the last element of an array.

Syntax:

array.pop()

Example:

var sample = [‘cat’, ‘dog’, ‘fish’];

sample.pop();

// Results: [‘cat’, ‘dog’]

Array.prototype.push()

Adds one or more elements to end of an array.

Syntax:

array.push([element1[, ...[, elementN]]])

Example:

var sample = [‘a’, ‘b’, ‘c’];

sample.push(‘d’, ‘e’);

//Results: [‘a’, ‘b’, ‘c’, ‘d’, ‘e’)

Accessor Methods

Does not modifies the target array

Array.prototype.concat()

Joining an array to another array or a value.

Syntax:

var new_array = old_array.concat(value1[, value2[, ...[, valueN]]])

Example:

var sample1 = [1, 2]

var sample2 = [3, 4];

var sample3 = sample1.concat(sample2)

console.log(sample3)

//Results: [1, 2 , 3, 4]

Array.prototype.indexOf()

Gets the index of an element from the array, and returns -1 if the index is not found or undefined.

Syntax:

array.indexOf(searchElement[, fromIndex])

Example:

var sample = [8, 17, 9, 20, 33]

sample.indexOf(20);

//Results: 3

sample.indexOf(100);

//Results: -1

Array.prototype.lastIndexOf()

Syntax:

array.lastIndexOf(searchElement)

array.lastIndexOf(searchElement, fromIndex)

Example:

var sample = [100, 50, 25, 30, 100];

sample.lastIndexOf(100);

//Result: 4

Array.prototype.toString()

Returns the array into a string representation, and returns it.

Syntax:

array.toString()

Example:

var sample = [‘Hello’, ‘World’, ‘Hi’, ‘Universe’];

sample.toString();

//Result: "Hello,World,Hi,Universe"

Iterator Methods

Array.prototype.forEach()

Executes a given function on every element of an array once.

Syntax:

arr.forEach(function callback(currentValue, index, array) {//your iterator}[, thisArg]);

Example:

var sample = ['house', 'park', 'mall'];

sample.forEach(function(element) {

console.log(element);

});

// Result : park

// Result : mall

JAVASCRIPT VARIABLE

Variable declarations, wherever they occur, are processed before any code is executed. The scope of a variable declared with var is its current execution context, which is either the enclosing function or, for variables declared outside any function, global. If you re-declare a JavaScript variable, it will not lose its value.

Declaring and initializing two variables

var a = 0, b = 0;

Assigning two variables with single string value

var a = 'A';

var b = a;

// Equivalent to:

var a, b = a = 'A';

let allows you to declare variables that are limited in scope to the block, statement, or expression on which it is used. This is unlike the var keyword, which defines a variable globally, or locally to an entire function regardless of block scope.

The const declaration creates a read-only reference to a value. It does not mean the value it holds is immutable, just that the variable identifier cannot be reassigned. For instance, in case the content is an object, this means the object itself can still be altered.

DOM (Document Object Model)

DOM(Document Object Model), according to Mozilla Developer Network, is a programming interface for HTML and XML documents. “It provides a structured representation of the document and it defines a way that the structure can be accessed from programs so that they can change the document structure, style and content.”

DOM Standards

  • W3C DOM
  • WHATWG DOM

For example, the standard DOM specifies that the getElementsByTagName method in the code below must return a list of all the <P> elements in the document:

HTML:

<p> The quick brown fox jumped over the lazy dog.</p>

<p> Clean clams crammed in clean cans.</p>

JAVASCRIPT:

var paragraphs = document.getElementsByTagName(“p”);

//Paragraph[0] is the paragraph containing “Hello World”

//Paragraph[1] is the paragraph containing “This is a paragraph”

Paragraph[0]

//Result :<p> The quick brown fox jumped over the lazy dog.</p>

paragraphs[1].nodeName

//Result :“P”

paragraphs[1].innerText

//Result :“Clean clams crammed in clean cans.”

Important Data types

There are a number of different data types that is being passed around the API that you should be aware of. These are the following important data types:

  • Document
    • When an object of type document is returned, this object will be the root document itself.
  • Element
    • This refers to the node of type element returned by a member of the DOM API. a method returns an element that is created in the DOM.
  • nodeList
    • An array of elements that are accessed by index in either:
      • list.item(1)
      • list[1]
  • Attribute
    • These are nodes in the DOM that when returned as an object, exposes special interface for attributes
  • namedNodeMap
    • Similar to an array but the items in it are accessed by name or by index.
