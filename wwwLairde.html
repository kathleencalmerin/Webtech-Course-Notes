Architecture of the World Wide Web

History

History

In 1989, Tim Berners-Lee had suggested a way to let all users(scientists) browse each others’ paper on the Internet. Because of that, he came up with the HTML, URLs, and HTTP.

How does the World Wide Web Works?

A Web browser sends a request to a server through a network, then the server responds to the request.


Web Browser – an application for retrieving, presenting, and traversing information resources


Web Server - a program/machine that accepts HTTP requests and return HTTP responses with optional data content.


Network- consists of two or more computers that are linked in order to share resources , exchange files, or allow electronic communications.
Foundations of the World Wide Web

HTML(Hypertext Mark-up Language) – Document layout language that defines the structure and content of Web Pages

CSS(Cascading Stylesheets)- describes how HTML elements are to be displayed on screen, paper, or in other media.

URI - is the way you identify any of those points of content, whether it be a page of text, a video or sound clip, a still or animated image, or a program. The most common form of URI is the Web page address, which is a particular form or subset of URI called a Uniform Resource Locator

HTTP – A request/response standard of a client and a server.

-Request/response messages consists of: Request/Status Line, Headers empty line,message body
